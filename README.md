# ホジリヌス

**鼻くそを取って遊ぼう！**

デカい鼻くそをひり出す快感と、思いっきり飛ばす快感。
常識を身につけ鼻をほじることを恥ずかしいものとしてしなくなってしまった、
子供心を忘れた全ての大人たちへ。

▶ **[遊ぶ](https://tomosan-7216.github.io/hojirinus/)**

## 遊びかた

スマホ縦画面想定。マウスでもタッチでも遊べる。

| 画面 | 場所 | やること |
|---|---|---|
| **鼻ほじり** | 中央（最初の画面） | 鼻の穴に指を入れて長押し。溜まったら離す |
| **鼻くそ飛ばし** | 上 | 在庫から選んでデコピンで飛ばす |
| **鼻くそ図鑑** | 右 | 集めた鼻くその形・説明・在庫を見る |
| **ショップ** | 左 | コインでステータスを強化する |
| **リザルト** | 下 | 今回のセッションの記録を見て終わる |

移動のしかたは2通り。**画面のふちに出ているタブを押す**か、**ふちに寄って押し込む**（隣のシーンが少し覗いて、そのまま押し込むと移動する）。
ほじっている最中や飛ばしている最中は、タブが消えて、ふちに触れても移動しない。

### 3つのコツ

1. **指を動かさないとゲージは溜まらない**。押しているだけでは1ミリも進まない。グリグリ動かした量で溜まる。激しく動かせば約2.4秒、ちまちま動かすと約6秒。ショップの「ほじりスピード」を上げると少ないグリグリで済む。
2. **溜まりきる前に離すと「スカ」**。何も出ない。
3. **デカい鼻くそはパワーが足りないと本気を出せない**。ショップの「想定飛距離」が赤字のサイズは、まだ本来の飛距離が出ていない。強化すると赤字が解ける。

## 使っているWeb API

外部のHTTP APIは使っていない（静的サイトなのでAPIキーが露出するため）。
代わりに、ブラウザが持つWeb APIで全部作っている。**画像ファイルも音声ファイルも1つも無い。**

| Web API | 用途 |
|---|---|
| **Canvas API** | 鼻・指・鼻くその描画すべて。鼻くその形は seed から手続き生成している |
| **Web Audio API** | SEを全部その場で合成。「ポン！」は急降下するサイン波＋バンドパスを通したノイズ |
| **Web Storage API** | 図鑑・在庫・コイン・強化・自己ベストの保存 |
| **Pointer Events API** | マウスとタッチを1系統で処理 |
| **Vibration API** | 溜め中に端末を震わせる（対応端末のみ） |
| **ResizeObserver API** | 画面サイズへのフィット（初期化・リサイズ・回転を1本で拾う） |
| **requestAnimationFrame** | メインループ |

## 動かす

### いちばん簡単：`hojirinus.html` 1個だけあればいい

**[hojirinus.html](hojirinus.html) をダウンロードしてダブルクリックするだけ。** CSSもJSも全部この中に入っているので、他のファイルは一切要らない。フォルダごと持ち歩く必要もサーバーも要らない。

> `index.html` **単体では動かない**ので注意。あれは隣に `style.css` と `src/`（またはバンドル）がある前提のファイルで、1個だけ取り出すと何も読めずに止まる。1ファイルで渡したいときは `hojirinus.html` のほう。

### フォルダごと使う場合

`index.html` をダブルクリックすればいい。こちらは `src/` の中身をそのまま読むので、コードをいじりながら動かすのに向く。

### なぜ2通りの読み込み方があるのか

`index.html` はまず `src/` の ES Modules を読もうとする。これが本体。
ただし **`file://` で開くと Chrome は `<script type="module">` を CORS で拒否する**（モジュールは origin `null` からフェッチできない）。ダブルクリックした瞬間これに当たり、何も動かない画面になる。

クラシックスクリプトはフェッチに CORS がかからないので、モジュールの読み込みに失敗したときだけ `hojirinus.bundle.js`（`src/` を1本に束ねた版）に自動で切り替える。

| 開き方 | 動くもの | 必要なファイル |
|---|---|---|
| GitHub Pages / ローカルサーバー（http） | `src/` の ES Modules（本体） | フォルダ一式 |
| `index.html` をダブルクリック（file://） | `hojirinus.bundle.js`（自動で切り替わる） | フォルダ一式 |
| `hojirinus.html` をダブルクリック | 全部インライン | **これ1個だけ** |

画像も音声も持っていないので、フェッチするものが他に無く、`file://` でも欠けるものはない。

### ビルドし直す

**`src/` を編集したら、必ず実行すること。** 忘れると `hojirinus.html` とダブルクリック版だけ古いまま残る（http版は常に最新なので、そちらでは気づけない）。

```powershell
powershell -ExecutionPolicy Bypass -File build.ps1
```

`build.ps1` が2つ吐く。node も python も要らない（Windows の PowerShell だけで完結する）。

- `hojirinus.bundle.js` … `src/` を依存順に結合し、`import`/`export` を差し替えたもの
- `hojirinus.html` … `index.html` に CSS とバンドルを流し込んだ1ファイル版

`$order` への記載漏れ、依存順の誤り、インライン化を壊す文字列は、ビルド時に例外で止まる。

## 設計メモ

- **図鑑（`seen`）と在庫（`stock`）は別テーブル。** 飛ばすと `stock` だけが減り、`seen` は絶対に触らない。「見たことがある」記録は永久に残る。
- **鼻くその形は seed から決まる。** だから未所持でもシルエットが描けて、しかも実物と必ず一致する。
- **飛距離は実際の放物線** `d = v²sin(2θ)/g`。45°が最適なのは物理からタダで出ている。
- **`effBonus = 1 + (sizeBonus - 1) × powerRatio`。** パワーが足りないほど、大きさの旨みが1.0に潰れる。ただし下限は1.0で、小より下回ることはない。デカい鼻くそは常に嬉しく、ただし本気を出せていない、という状態にしてある。
