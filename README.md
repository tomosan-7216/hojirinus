# ホジリヌス

**鼻くそを取って遊ぼう！**

デカい鼻くそをひり出す快感と、思いっきり飛ばす快感。
常識を身につけ鼻をほじることを恥ずかしいものとしてしなくなってしまった、
子供心を忘れた全ての大人たちへ。

## 遊びかた

スマホ縦画面想定。マウスでもタッチでも遊べる。

| 画面 | 場所 | やること |
|---|---|---|
| **鼻ほじり** | 中央（最初の画面） | 鼻の穴に指を入れて長押し。溜まったら離す |
| **鼻くそ飛ばし** | 上 | 在庫から選んでデコピンで飛ばす |
| **鼻くそ図鑑** | 右 | 集めた鼻くその形・説明・在庫を見る |
| **ショップ** | 左 | コインでステータスを強化する |
| **リザルト** | 下 | 今回のセッションの記録を見て終わる |

**画面のふちに寄ると、隣のシーンが少し覗いて、そのまま押し込むと移動する。**
ほじっている最中や飛ばしている最中は、ふちに触れても移動しない。

### 3つのコツ

1. **溜まりきる前に離すと「スカ」**。何も出ない。
2. **グリグリは、してもしなくてもゲージの速度は変わらない**。ただしグリグリしたぶんだけ、鼻の穴が歪み、画面が揺れ、抜けるときの音が太くなり、鼻くそが勢いよく飛び出す。得はしないが、気持ちがいい。
3. **デカい鼻くそはパワーが足りないと本気を出せない**。ショップの「想定飛距離」が赤字のサイズは、まだ本来の飛距離が出ていない。強化すると赤字が解ける。

## 使っているWeb API

外部のHTTP APIは使っていない（静的サイトなのでAPIキーが露出するため）。
代わりに、ブラウザが持つWeb APIで全部作っている。**画像ファイルも音声ファイルも1つも無い。**

| Web API | 用途 |
|---|---|
| **Canvas API** | 鼻・指・鼻くその描画すべて。鼻くその形は seed から手続き生成している |
| **Web Audio API** | SEを全部その場で合成。「ポン！」は急降下するサイン波＋バンドパスを通したノイズ |
| **Web Storage API** | 図鑑・在庫・コイン・強化・自己ベストの保存 |
| **Pointer Events API** | マウスとタッチを1系統で処理 |
| **Vibration API** | 溜め中に端末を震わせる（対応端末のみ） |
| **ResizeObserver API** | 画面サイズへのフィット（初期化・リサイズ・回転を1本で拾う） |
| **requestAnimationFrame** | メインループ |

## 動かす

ビルド不要。ただし **ES Modules は `file://` では動かない**（CORSで弾かれる）ので、httpで配信する必要がある。

```bash
# 何かしらの静的サーバーで配信する
npx serve .
# → http://localhost:3000
```

VS Code なら Live Server 拡張でも可。
GitHub Pages は http 配信なので、公開版はそのまま動く。

## 設計メモ

- **図鑑（`seen`）と在庫（`stock`）は別テーブル。** 飛ばすと `stock` だけが減り、`seen` は絶対に触らない。「見たことがある」記録は永久に残る。
- **鼻くその形は seed から決まる。** だから未所持でもシルエットが描けて、しかも実物と必ず一致する。
- **飛距離は実際の放物線** `d = v²sin(2θ)/g`。45°が最適なのは物理からタダで出ている。
- **`effBonus = 1 + (sizeBonus - 1) × powerRatio`。** パワーが足りないほど、大きさの旨みが1.0に潰れる。ただし下限は1.0で、小より下回ることはない。デカい鼻くそは常に嬉しく、ただし本気を出せていない、という状態にしてある。

詳しくは [docs/実装計画書.md](docs/実装計画書.md)。
