## 2. 企画書の投入

**入れたプロンプト（要旨）**
「ホジリヌス」の企画書全文。要点は以下。

- コンセプト：鼻くそを取って遊ぼう！　デカい鼻くそをひり出す快感と、思いっきり飛ばす快感。
- ほじって集めて飛ばす。図鑑・在庫・コイン・強化。明確なゴールは無く、セッション型。
- **図鑑は「見たことがある」記録として永久に残り、実体は別に在庫として個数を持つ。飛ばすと在庫は減るが図鑑は消えない。**
- 画面はスマホ縦。中央にほじりゾーンを置き、四方（上=飛ばし／右=図鑑／左=ショップ／下=やめる）にシーンを配置。画面端に触れるとシームレスにスライド遷移。**アクション中はエッジ判定を取らない。**
- ほじりゲージは**グリグリしてもしなくても同じ速度**（2〜3秒）。慣れたプレイヤーのテンポを優先するため。
- 鼻くそはレア度とは別に大きさ（小・中・大・特大・伝説）を持つ。大きいほど飛ぶが、パワーが要る。序盤はパワー上限が低く、デカい鼻くそは本来の飛距離を出せない。
- 飛ばしはデコピン。角度をメトロノームで決め（ベスト45度）、長押しでゲージを溜める。**タイミングを計るものではなく、思い切り溜める快感を味わってほしい**ので満タン後も押し続けられる。満タンかつ40〜45度でヒットストップ。
- 未確定点として3つ提示：①エッジ発火方式 ②強化後の即時フィードバック ③飛ばしシーンの「世界」

**AIの応答（要旨）→ [実装計画書.md](実装計画書.md) を出力**

技術選定・データ設計・数式・フェーズを固めたうえで、企画に対して以下の判断を返してきた。

1. **「このゲームの本体は演出である」と位置づけた。**
   ほじりのメカニクスを分解すると「穴に指を入れて2.5秒待って離す」しかなく、ゲージ速度をグリグリと無関係にした時点でプレイヤー入力が結果に影響する余地が意図的にゼロになっている。だから面白さは抽選の引きと手応えだけで支えられており、演出は仕上げの磨きではなく機能そのもの、と。工数の山もそこに置く前提で組まれた。
2. **グリグリの扱い。** 損得ゼロだと気づいたプレイヤーがやめてしまう懸念に対し、報酬を足すと企画のテンポ判断を壊すので、快感側（歪み・揺れ・音の太さ・飛び出す勢い）だけで解決する方針を明記。
3. **鼻くその形状を seed からの手続き生成にした。** これで「未所持でもシルエットは図鑑に出る」という要求が自動的に満たされる（形がseedから計算できるので、シルエットと実物が必ず一致する）。画像アセットもゼロになった。
4. **飛距離を実物理 `d = v²sin(2θ)/g` で出した。** 企画指定の「ベストは45度」が物理からタダで成立し、40度=sin(80°)=98.5% なのでヒットストップ窓40〜45度とも噛み合う。
5. **パワー不足のデカい鼻くそを、小より下回らせない**設計判断。`effBonus = 1 + (sizeBonus-1) × powerRatio` とし、下限を1.0に。下回らせると「デカいの引いたのに損」になり、企画の芯である『デカい鼻くそをひり出す快感』と正面衝突するため。
6. **未確定点3つへの回答**
   - ①エッジ：帯に入ると隣が12%覗き、奥まで押すか少し留まると確定。覗きがチュートリアルを兼ねるので説明文が要らなくなる。
   - ②強化フィードバック：+0.5%は単体では絶対に体感できないので、数字を大きくするのではなく「長押し連打購入（SEのピッチが上がる）」「10%ごとのマイルストーンで指が変化」「想定飛距離の常時表示（パワー不足は赤字→強化で解ける）」で埋める。
   - ③飛ばしの世界：**風・的・障害物を入れないことを推奨。** どれもRNGか正解タイミングを持ち込み、企画が明確に否定した「タイミングを計るものではない」と衝突して、ゲージが"計るもの"に戻ってしまうため。代わりに背景の段階変化（机→部屋→街→成層圏→宇宙）とサイズ別自己ベスト5本に寄せた。
   - おまけ：画面下の「やめる」はWebにプロセス終了が無いので**セッションリザルト**に読み替え。企画の「気が済むまで遊んで辞める」に終わりの一拍を与えるため。

## 4. 実装中に起きたこと（記録）

AIが自分でブラウザ上で動かして直したもの。指示ではなく、AIが検証して見つけた。

| 症状 | 原因 | 対処 |
|---|---|---|
| 開発PCにnode/pythonが無く、ES Modulesがfile://で動かせない | Microsoft Store版pythonはスタブで実体なし | .NETの`HttpListener`で依存ゼロの簡易サーバーを自作して検証した |
| `#stage`が`scale(0)`のまま固まる | 初回`doFit()`がレイアウト確定前に走っていた | `ResizeObserver`に変更（初期化・リサイズ・回転を1本で拾える） |
| **クリック位置が実際の座標とズレる／キャンバス右側が切れる** | `#stage`がflexアイテムとして720px→460pxに**縮められて**いた。内部の720x1280座標系がズレ、絶対配置のキャンバスが溢れて`overflow:hidden`で切れていた | `flex: 0 0 auto` を追加 |
| 初登場アイリスが実時間で走る | `setTimeout`で発火していた | ゲームループ側（ゲーム時間）に移動。ヒットストップやフレーム落ちとズレなくなった |
| 鼻くその形がヒトデになる | 形状生成のサイン波の振幅を欲張りすぎ | 合計振幅を半径の3割までに制限 |
| 顔が「卵」に見える | 楕円で描いた縁が見えていた | 画面いっぱいに塗って極端なクローズアップにし、ビネットで鼻に目を寄せた |
| `touch-action:none`で図鑑が縦スクロールできなくなる懸念 | touch-actionは「自分と祖先の交差」で決まる | `#stage`には置かず、キャンバスシーンにだけ効かせた |
